TRAILING STOP — STATE MACHINE (4 estados):
  WATCHING:   precio < entrada * (1 + breakeven_trigger)  [default: 5%]
  BREAKEVEN:  SL = precio_entrada (proteger capital)
  TRAILING:   SL = max(SL_anterior, precio * (1 - trailing_pct))  [default: 3%]
  CLOSING:    precio <= SL → ejecutar SmartExit

FILTRO DE LIQUIDEZ PRE-TRADE (calibrado para no ser excesivamente restrictivo):
  spread_pct = (best_ask - best_bid) / mid_price * 100
  depth_5 = sum(qty top 5 niveles bid + ask)
  slippage_est = position_size_usd / depth_5 * 100

  Rechazar si:
    spread_pct > config.max_spread          [default calibrado: 6%, no 3%]
    slippage_est > config.max_slippage      [default calibrado: 3%, no 1%]
    depth_5 < config.min_depth_usd          [default calibrado: $500, no $2000]

PIPELINE COMPLETO (clase NewsFeedPipeline):

  async def run_forever(self, active_markets):
    while True:
      await self.fetch_and_score(active_markets)
      await asyncio.sleep(60)  # cada 60s para mercados activos

  async def fetch_and_score(self, active_markets):
    for category, feeds in RSS_FEEDS.items():
      for feed_url in feeds:
        try:
          feed = feedparser.parse(feed_url)
          for entry in feed.entries[:10]:  # últimas 10 por feed
            title = entry.get("title", "")
            h = hashlib.md5(title.encode()).hexdigest()
            if h in self.seen_hashes: continue
            self.seen_hashes.add(h)

            # PASO 1: Filtro de relevancia con keywords del mercado
            relevant_markets = self.find_relevant_markets(title, active_markets)
            if not relevant_markets: continue

            # PASO 2: VADER scoring (offline, sin límites, instantáneo)
            score = self.vader.polarity_scores(title)["compound"]

            for market_id in relevant_markets:
              prev_score = self.market_scores.get(market_id, 0.0)
              self.market_scores[market_id] = score
              shift = abs(score - prev_score)

              # UMBRAL CALIBRADO: 0.25 no 0.50 — genera señales reales
              if shift > config.sentiment_shift_threshold:  # default: 0.25
                # PASO 3: Escalar a IA (Groq) para análisis profundo
                await self.analyze_with_ai(title, market_id, score, shift)
        except Exception as e:
          logger.warning("feed_error", url=feed_url, error=str(e))

  def find_relevant_markets(self, title, markets):
    """Matching de keywords — ni demasiado estricto ni demasiado laxo"""
    title_lower = title.lower()
    relevant = []
    for market in markets:
      # Keywords del mercado definidas al crearlo en Polymarket
      # Ej: mercado "¿Ganará Harris?" → keywords: ["harris", "democrat", "election"]
      score = sum(1 for kw in market.keywords if kw in title_lower)
      if score >= 1:  # al menos 1 keyword match (no exigir 2+, muy restrictivo)
        relevant.append(market.id)
    return relevant

4 REGÍMENES (detectar cada 4h usando datos históricos del mercado):

  TRENDING_UP:
    Condición: > 60% de las velas de 1h son alcistas en últimas 12h
               AND slope_linreg(probabilidades_24h) > 0.002/h
    Estrategias activas: MOMENTUM (principal), MEAN_REVERSION (desactivada)
    Ajustes: SL más amplio (+1.5%), TP más lejano (+3%), size normal

  TRENDING_DOWN:
    Condición: > 60% de velas bajistas AND slope < -0.002/h
    Estrategias activas: MOMENTUM inverso
    Ajustes: mismos que TRENDING_UP pero dirección opuesta

  RANGING:
    Condición: precio oscila ±2% alrededor de media en 12h
               AND Bollinger Band width < 0.04
    Estrategias activas: MEAN_REVERSION (principal), LIQUIDITY_CAPTURE
    Ajustes: SL más cercano (-0.5%), TP más cercano (-2%), size normal

  HIGH_VOLATILITY:
    Condición: ATR(14) > 2x ATR promedio de los últimos 30 días
               OR spread promedio del mercado > 2x su promedio histórico
    Estrategias activas: NINGUNA — modo observación
    Ajustes: reducir size todas las posiciones 50%, no abrir nuevas
    EXCEPCIÓN: si hay señal de IA con confidence > 0.80 → permitir tamaño 30%

class MarketRegimeDetector:
  def detect(self, market_id: str, price_history: list) -> Regime:
    closes = [p.probability for p in price_history[-24:]]  # 24h de datos

    # Slope de tendencia
    n = len(closes)
    slope = (closes[-1] - closes[0]) / max(n, 1)

    # Bollinger Band width
    mean = statistics.mean(closes)
    std = statistics.stdev(closes) if len(closes) > 1 else 0
    bb_width = (2 * std) / mean if mean > 0 else 0

    # ATR simplificado
    ranges = [abs(closes[i] - closes[i-1]) for i in range(1, len(closes))]
    atr = statistics.mean(ranges[-14:]) if len(ranges) >= 14 else 0
    atr_avg_30d = self.get_avg_atr(market_id, days=30)

    if atr > atr_avg_30d * 2:
      return Regime.HIGH_VOLATILITY
    elif slope > 0.002 and sum(1 for i in range(1, n) if closes[i] > closes[i-1]) > n * 0.6:
      return Regime.TRENDING_UP
    elif slope < -0.002 and sum(1 for i in range(1, n) if closes[i] < closes[i-1]) > n * 0.6:
      return Regime.TRENDING_DOWN
    else:
      return Regime.RANGING

ESTRATEGIA 1 — MEAN REVERSION (mejor en régimen RANGING):
  Señal de entrada:
    z_score = (prob_actual - prob_media_7d) / prob_std_7d
    if abs(z_score) >= config.zscore_threshold:  # default CALIBRADO: 1.8 no 2.5
      direction = "BUY" if z_score < -1.8 else "SELL"

  Filtro de confirmación (aplicar en orden, NO todos a la vez — eso mata las señales):
    1. Régimen == RANGING (obligatorio para esta estrategia)
    2. Sentiment de Groq no contradice la dirección (opcional, si Groq disponible)
    3. Volumen > config.min_volume  [default: $1,000, no $10,000]

  Salida: z_score regresa a [-0.5, 0.5] OR trailing stop activado

  NOTA DE CALIBRACIÓN: Con z_score >= 2.5 este mercado generaría ~1 señal/semana.
  Con z_score >= 1.8 genera ~2-3 señales/semana en mercados activos.

ESTRATEGIA 2 — MOMENTUM (mejor en régimen TRENDING):
  Señal de entrada:
    cambio_1h = prob_actual - prob_hace_1h
    volumen_1h vs volumen_promedio_7d
    if abs(cambio_1h) >= 0.025 AND volumen_1h > volumen_promedio * 1.5:
      direction = "BUY" if cambio_1h > 0 else "SELL"

  Confirmación adicional (opcional):
    if ai_analysis.direction == direction AND ai_analysis.confidence >= 0.60:
      signal_strength += 0.2  # boost en sizing con Kelly

  Salida: momentum pierde fuerza (cambio_1h reciente < 0.01) OR trailing stop

ESTRATEGIA 3 — LIQUIDITY CAPTURE (market making, régimen RANGING):
  Solo en mercados con: volumen_diario > $5,000 Y spread < 4%
  Colocar órdenes limit bid/ask con spread propio de 1-2%
  Gestión de inventario: si posición neta > 20% del size máximo → neutralizar
  ⚠️ Verificar en investigación (Módulo 1) si CLOB permite actualizar órdenes

ESTRATEGIA 4 — CROSS-MARKET ARBITRAGE:
  Trigger: cuando Módulo 11 detecta divergencia > 8% vs Metaculus/PredictIt
  Entrar en dirección del mercado externo más informado
  Salida: divergencia se reduce a < 3% OR trailing stop

UMBRALES DE ACTIVACIÓN (balanceados — no tan estrictos que bloqueen todo):
  ✓ Sharpe >= 0.4   (no 0.5 — con pocos datos es difícil superar 0.5)
  ✓ MaxDD <= 35%    (no 30% — prediction markets son volátiles por naturaleza)
  ✓ Trades >= 20    (no 30 — 90 días pueden no dar 30 trades con umbrales correctos)

class SmartRateLimiter:
  LIMITS = {
    "get_orderbook": 60,   # req/min
    "place_order":   30,
    "get_positions": 30,
    "get_markets":   20,
  }
  RESERVE = 0.20  # reservar 20% de cuota como buffer

  async def call(self, endpoint: str, fn: Callable):
    limit = self.LIMITS[endpoint] * (1 - self.RESERVE)
    current = self.calls_this_minute.get(endpoint, 0)
    if current >= limit:
      await asyncio.sleep(60 / limit)
    result = await fn()
    self.calls_this_minute[endpoint] = current + 1
    return result

Alerta si endpoint supera 70% de cuota en < 1h

SIMULADOR DE SLIPPAGE REALISTA (paper mode):
  Mantener distribución histórica de slippage por mercado (histograma)
  Al simular: samplear de la distribución, NO usar valor fijo
  Ajustar por tamaño: slippage_final *= min(2.0, size / avg_order_size)
  Resultado: paper trading más cercano a ejecución real

A/B TESTING: Pool A vs Pool B (50/50), mínimo 14 días o 20 trades por pool
  Ganadora si: p-value < 0.10 Y diferencia > 8% en Sharpe (calibrado, no 0.05/10%)
  Si no hay diferencia: mantener el más conservador

MULTI-WALLET: hasta 3 wallets con estrategias y capital independientes
  Dashboard unificado con PnL consolidado y comparación de performance
