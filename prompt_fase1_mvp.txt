MÓDULO 2: MOTOR DE TRADING CORE (TradingEngine)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

REGLAS DE INTEGRIDAD (hardcoded):
• Una posición activa por market_id / conditionId
• Verificación de duplicados antes de CUALQUIER orden
• Circuit breaker: stop si pérdida > MAX_DAILY_LOSS_PCT en 24h
• Nunca operar si balance < capital_mínimo_configurado
• Nunca operar si latencia API > 8s (modo degradado)
• Queue de prioridad: cancelaciones > modificaciones > nuevas órdenes

TRAILING STOP — STATE MACHINE (4 estados):
  WATCHING:   precio < entrada * (1 + BREAKEVEN_TRIGGER)
  BREAKEVEN:  SL = precio_entrada (proteger capital)
  TRAILING:   SL = max(SL_anterior, precio * (1 - TRAILING_PCT))
  CLOSING:    precio <= SL → ejecutar cierre (market order en esta fase)

FILTRO DE LIQUIDEZ PRE-TRADE:
  spread_pct = (best_ask - best_bid) / mid_price * 100
  depth_5 = sum(qty top 5 niveles bid + ask)
  slippage_est = position_size_usd / depth_5 * 100

  Rechazar si spread_pct > MAX_SPREAD_PCT
             OR slippage_est > MAX_SLIPPAGE_PCT
             OR depth_5 < MIN_DEPTH_USD

  Loggear siempre el motivo del rechazo con valores concretos.

COMISIONES:
  PAPER: PAPER_FEE_PCT entrada + salida
  REAL:  getFeeRateBps() en tiempo real — nunca asumir valor fijo

ASYNC: asyncio + aiohttp | timeout: 10s | retry backoff: 2s/8s/30s

STACK GRATUITO:
  VADER: pip install vaderSentiment → offline, sin límite, instantáneo
         IMPORTANTE: Solo funciona con texto en INGLÉS
  Groq:  pip install groq → 14,400 req/día gratis (tier free)
         Modelo: llama-3.1-70b-versatile
         Uso: análisis profundo solo cuando VADER detecta shift > SENTIMENT_SHIFT

Clase AIAnalyzer:
  - daily_limit conservador: 300 de 14,400 disponibles
  - Si se agota Groq → fallback a VADER puro
  - Prompt para Groq: pedir JSON con direction, confidence, impact, should_trade
  - Si confidence < AI_CONFIDENCE_MIN → should_trade = False
  - NOTA IMPORTANTE: La "confidence" de un LLM NO es una probabilidad calibrada.
    Los valores tienden a concentrarse en 0.6-0.8. Tener esto en cuenta al
    interpretar señales — no tratar como probabilidad matemática exacta.

Fallback VADER:
  direction = "UP" si score > 0.1, "DOWN" si < -0.1, "NEUTRAL" si no
  should_trade = abs(score) > 0.30

MÓDULO 5: ESTRATEGIA — MEAN REVERSION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Clase BaseStrategy (abstracta):
  generate_signal(market_data) -> Signal | None
  calculate_size(signal, portfolio_state) -> float

MEAN REVERSION:
  Señal de entrada:
    z_score = (prob_actual - prob_media_7d) / prob_std_7d
    if abs(z_score) >= ZSCORE_THRESHOLD:
      direction = "BUY" si z_score < -ZSCORE_THRESHOLD, "SELL" si no

  LIMITACIÓN DEL Z-SCORE: Las probabilidades en prediction markets NO siguen
  distribución normal (bounded [0,1], saltos discontinuos con noticias).
  El z-score es una heurística útil, no una medida estadística exacta.

  Filtros de confirmación (en orden, parar al primer rechazo):
    1. Volumen > MIN_DAILY_VOLUME
    2. Filtro de liquidez pre-trade (Módulo 2)
    3. Si IA disponible: sentiment no contradice dirección (opcional)

  Salida: z_score regresa a [-0.5, 0.5] OR trailing stop activado

  Loggear SIEMPRE señales generadas Y rechazadas con motivo y valores.

NOTA: Esta es la única estrategia en Fase 1. Momentum y otras se añaden en Fase 2.

MÓDULO 6: GESTIÓN DE PORTAFOLIO (básica)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SIZING (conservador hasta tener datos):
  Si < 50 trades históricos: usar DEFAULT_POSITION_PCT fijo
  Si >= 50 trades: calcular Kelly criterion:
    kelly_full = (win_rate * avg_win - (1-win_rate) * avg_loss) / avg_win
    kelly_fraction = kelly_full * 0.25  (Quarter Kelly)
    position_size = capital * min(kelly_fraction, MAX_POSITION_PCT)

ANTI-MARTINGALE:
  consecutive_losses = contar pérdidas consecutivas
  recovery_factor = max(0.30, 1.0 - consecutive_losses * 0.15)
  Si consecutive_losses >= 5: PAUSA 24h + alerta

LÍMITES POR CATEGORÍA:
  política: 40% | crypto: 30% | deportes: 20% | otros: 30%
  Hardcoded, no configurables por el usuario
